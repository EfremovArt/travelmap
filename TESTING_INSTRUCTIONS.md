# Инструкция по тестированию коммерческих постов

## Шаги для тестирования:

### 1. Создание таблицы в базе данных
Выполните SQL скрипт `commercial_posts_table.sql` в вашей базе данных MySQL.

### 2. Добавление тестовых данных
Выполните SQL скрипт `test_commercial_posts.sql` для добавления тестовых коммерческих постов.

### 3. Запуск приложения
Запустите Flutter приложение и перейдите в раздел "Альбомы".

### 4. Проверка индикаторов
- Найдите альбом с ID 10 (или любой другой альбом, для которого вы добавили коммерческие посты)
- На обложке альбома должен появиться **оранжевый индикатор** с числом коммерческих постов
- Индикатор должен иметь **пульсирующий эффект**

### 5. Просмотр коммерческих постов
- **Нажмите на оранжевый индикатор** на обложке альбома
- Должен открыться экран "Коммерческие предложения"
- Вы должны увидеть список коммерческих постов с:
  - Изображениями (если есть)
  - Заголовками
  - Описаниями
  - Ценами
  - Контактной информацией
  - Информацией об авторах

### 6. Создание нового коммерческого поста
- В экране коммерческих постов нажмите кнопку **"+"** в правом верхнем углу
- Заполните форму:
  - Заголовок (обязательно)
  - Описание (необязательно)
  - Цена и валюта (необязательно)
  - Контактная информация (необязательно)
- Нажмите "Создать коммерческий пост"
- Новый пост должен появиться в списке

### 7. Проверка обновления индикатора
- Вернитесь к списку альбомов
- Индикатор на обложке должен обновиться с новым количеством постов

## Что должно работать:

✅ **Индикаторы на обложках** - оранжевые бейджи с количеством постов
✅ **Пульсирующий эффект** - привлекает внимание
✅ **Навигация** - нажатие на индикатор открывает список постов
✅ **Создание постов** - кнопка "+" для добавления новых постов
✅ **Отображение постов** - карточки с полной информацией
✅ **Обновление счетчиков** - автоматическое обновление после изменений

## Возможные проблемы:

❌ **Индикаторы не отображаются** - проверьте, что в базе данных есть коммерческие посты для альбома
❌ **Ошибка при создании поста** - проверьте, что серверные скрипты загружены и работают
❌ **Не отображается кнопка "+"** - убедитесь, что currentUserId передается корректно

## Отладка:

1. Проверьте логи в консоли Flutter
2. Убедитесь, что API endpoints доступны
3. Проверьте структуру базы данных
4. Убедитесь, что все файлы загружены на сервер

## API Endpoints для проверки:

- `GET /commercial/get_count.php?album_id=10` - должен вернуть количество постов
- `GET /commercial/get_posts.php?album_id=10` - должен вернуть список постов
- `POST /commercial/create_post.php` - должен создать новый пост
