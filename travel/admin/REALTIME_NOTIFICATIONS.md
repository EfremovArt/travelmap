# Система уведомлений в реальном времени

## Описание
Система уведомлений работает в реальном времени и автоматически обновляется без перезагрузки страницы. Уведомления исчезают после просмотра соответствующего контента.

## Как работает

### 1. Автоматическое обновление
- **Уведомления** обновляются каждые 30 секунд
- **Счетчики** обновляются каждые 2 минуты на странице модерации
- **Списки фото и комментариев** обновляются каждые 30 секунд
- При любом действии (удаление, просмотр) счетчики обновляются мгновенно
- Автообновление останавливается при переключении на другую вкладку браузера

### 2. Отслеживание просмотров
Система отслеживает два типа контента:
- **Фотографии** - новые фото на модерации
- **Комментарии** - новые комментарии

### 3. Автоматическое исчезновение уведомлений

#### При клике на уведомление:
- Клик на "Новые фото на модерации" → отмечает фото как просмотренные
- Клик на "Новые комментарии" → отмечает комментарии как просмотренные
- Уведомление исчезает через 0.5 секунды

#### При просмотре контента:
- Открытие вкладки "Фотографии" → через 1 сек отмечается просмотр
- Открытие вкладки "Комментарии" → через 1 сек отмечается просмотр
- Счетчик обнуляется автоматически

#### После действий:
- Удаление фото → обновление счетчиков и уведомлений
- Удаление комментария → обновление счетчиков и уведомлений
- Массовое удаление → обновление счетчиков и уведомлений

### 4. Визуальные индикаторы

#### Колокольчик уведомлений:
- **Красный бейдж** - есть новые уведомления (показывает количество)
- **Скрыт** - нет новых уведомлений
- **Анимация пульсации** - привлекает внимание к новым уведомлениям

#### Вкладки модерации:
- **Красный бейдж** - есть новые элементы
- **Серый бейдж** - нет новых элементов

## Технические детали

### API endpoints:
- `GET /api/moderation/get_new_counts.php` - получение счетчиков новых элементов
- `POST /api/moderation/mark_as_viewed.php` - отметка просмотра

### JavaScript функции:
- `loadNotifications()` - загрузка уведомлений
- `updateNotificationUI()` - обновление UI уведомлений
- `markNotificationAsViewed()` - отметка уведомления как просмотренного
- `updateCounts()` - обновление счетчиков на странице модерации
- `markAsViewed()` - отметка контента как просмотренного

### Интервалы обновления:
- Уведомления: 30 секунд
- Счетчики модерации: 2 минуты
- Списки фото: 30 секунд (только на активной вкладке)
- Списки комментариев: 30 секунд (только на активной вкладке)
- Бейдж модерации в сайдбаре: 2 минуты

### Оптимизация производительности:
- Автообновление работает только для активной вкладки (фото ИЛИ комментарии)
- При переключении вкладки браузера автообновление останавливается
- При возврате на страницу автообновление возобновляется

## Преимущества

1. **Реальное время** - не нужно обновлять страницу
2. **Умное отслеживание** - показывает только непросмотренное
3. **Автоматическое исчезновение** - уведомления пропадают после просмотра
4. **Низкая нагрузка** - оптимизированные интервалы обновления
5. **Интуитивность** - понятные визуальные индикаторы
