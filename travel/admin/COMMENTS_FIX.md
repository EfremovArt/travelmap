# Исправление отображения комментариев

## Проблема
Комментарии не отображались в профиле пользователя, хотя в базе данных они были.

## Причина
Неправильное имя колонки в SQL запросах. Использовалось `c.comment_text`, но в таблице `comments` колонка называется `c.comment`.

## Исправление

### Файл: `api/users/get_user_details.php`

**Изменено 2 SQL запроса:**

#### 1. Комментарии пользователя (написанные)
```sql
-- Было:
SELECT c.id, c.comment_text, c.created_at, ...

-- Стало:
SELECT c.id, c.comment as comment_text, c.created_at, ...
```

#### 2. Комментарии к постам пользователя (полученные)
```sql
-- Было:
SELECT c.id, c.comment_text, c.created_at, ...
WHERE p.user_id = :user_id

-- Стало:
SELECT c.id, c.comment as comment_text, c.created_at, ...
WHERE p.user_id = :user_id AND c.user_id != :user_id
```

**Дополнительно:** Добавлено условие `AND c.user_id != :user_id` чтобы исключить комментарии пользователя к своим же постам.

### Файл: `test_user_comments.php`
Исправлены те же SQL запросы для тестирования.

## Структура таблицы comments

```
id - int
user_id - int
photo_id - int
comment - text  ← правильное имя колонки
created_at - timestamp
```

## Проверка

1. Откройте `test_user_comments.php?user_id=20`
2. Должны отобразиться все комментарии пользователя
3. Откройте `user_details.php?id=20`
4. Перейдите на вкладку "Комментарии"
5. Проверьте обе подвкладки: "Написанные" и "Полученные"

## Результат

✅ Комментарии теперь отображаются корректно  
✅ "Написанные" показывает все комментарии пользователя  
✅ "Полученные" показывает комментарии от других к постам пользователя  
✅ Исключены дубликаты (комментарии пользователя к своим постам)
