# Функция управления платными постами

## Обзор

Добавлена новая вкладка "Платные посты" на странице детальной информации пользователя, которая позволяет просматривать и управлять коммерческими постами пользователя.

## Возможности

### 1. Просмотр платных постов

На вкладке "Платные посты" отображается:
- **Превью** - изображение поста
- **Название** - заголовок платного поста
- **Статус** - активен (зеленый) или неактивен (серый)
- **Тип** - фото, альбом или отдельный пост
- **Описание** - текст описания
- **Связь** - если пост связан с фото или альбомом, показывается название
- **Координаты** - если указаны GPS координаты
- **Дата создания** - когда был создан пост

### 2. Управление статусом

**Активация/деактивация:**
- Кнопка "Активировать" - делает пост видимым для пользователей
- Кнопка "Деактивировать" - скрывает пост, но не удаляет его
- Статус меняется мгновенно с визуальной обратной связью

### 3. Просмотр деталей

Кнопка "Детали" открывает полную страницу с информацией о платном посте:
- Все связанные фото (если это альбом)
- Полная информация о локации
- История изменений
- Статистика просмотров

### 4. Удаление

Кнопка "Удалить" позволяет полностью удалить платный пост:
- Запрашивается подтверждение
- Удаляются все связи (favorites, photo_commercial_posts)
- Сам пост удаляется из базы данных
- Статистика автоматически обновляется

## Визуальные индикаторы

### Статус поста
- 🟢 **Зеленая рамка** - активный пост
- ⚫ **Серая рамка** - неактивный пост

### Бейджи
- 🟢 **Активен** - зеленый бейдж
- ⚫ **Неактивен** - серый бейдж
- 🟡 **Тип поста** - желтый бейдж (Фото/Альбом/Отдельный)

### Кнопки
- 🔵 **Детали** - синяя кнопка для просмотра
- 🟡 **Деактивировать** - желтая кнопка для активных постов
- 🟢 **Активировать** - зеленая кнопка для неактивных постов
- 🔴 **Удалить** - красная кнопка для удаления

## Типы платных постов

### 1. Фото (photo)
- Связан с конкретным фото из таблицы photos
- Показывает превью этого фото
- При удалении платного поста, само фото остается

### 2. Альбом (album)
- Связан с альбомом из таблицы albums
- Показывает обложку альбома
- При удалении платного поста, альбом остается

### 3. Отдельный (standalone)
- Независимый платный пост
- Имеет собственное изображение (image_url)
- Не связан с другими сущностями

## API Endpoints

### GET /api/posts/get_all_commercial_posts.php
Получение списка платных постов пользователя

**Параметры:**
- `user_id` - ID пользователя (обязательный)
- `page` - номер страницы (по умолчанию 1)
- `per_page` - количество на странице (по умолчанию 50, макс 100)
- `type` - фильтр по типу (photo/album/standalone)
- `search` - поиск по названию/описанию

**Ответ:**
```json
{
  "success": true,
  "commercialPosts": [
    {
      "id": 1,
      "user_id": 123,
      "user_name": "Иван Иванов",
      "type": "photo",
      "title": "Красивый закат",
      "description": "Описание...",
      "preview": "https://...",
      "is_active": 1,
      "latitude": 55.7558,
      "longitude": 37.6173,
      "created_at": "2024-01-01 12:00:00",
      "related_title": "Название связанного фото"
    }
  ],
  "pagination": {
    "total": 10,
    "perPage": 50,
    "currentPage": 1,
    "lastPage": 1
  }
}
```

### POST /api/posts/toggle_commercial_status.php
Изменение статуса платного поста

**Параметры:**
- `commercial_id` - ID платного поста (обязательный)
- `is_active` - новый статус (0 или 1)

**Ответ:**
```json
{
  "success": true,
  "message": "Платный пост активирован",
  "is_active": 1
}
```

### POST /api/posts/delete_commercial_post.php
Удаление платного поста

**Параметры:**
- `commercial_id` - ID платного поста (обязательный)

**Ответ:**
```json
{
  "success": true,
  "message": "Платный пост успешно удален"
}
```

## Что удаляется при удалении платного поста

При удалении выполняется транзакция, которая удаляет:
1. Записи в `commercial_favorites` (если есть)
2. Записи в `photo_commercial_posts` (если есть)
3. Сам пост из `commercial_posts`

**Важно:** Связанные фото и альбомы НЕ удаляются!

## Безопасность

- Все операции требуют авторизации администратора
- Используются транзакции для целостности данных
- Подтверждение перед удалением
- Валидация всех входных параметров

## Использование

### Просмотр платных постов пользователя
1. Откройте страницу пользователя
2. Перейдите на вкладку "Платные посты"
3. Просмотрите список всех коммерческих постов

### Активация/деактивация поста
1. Найдите нужный пост в списке
2. Нажмите кнопку "Активировать" или "Деактивировать"
3. Статус изменится мгновенно
4. Счетчик активных постов обновится

### Просмотр деталей
1. Нажмите кнопку "Детали" рядом с постом
2. Откроется новая вкладка с полной информацией
3. Можно просмотреть все связанные данные

### Удаление платного поста
1. Нажмите кнопку "Удалить"
2. Подтвердите действие в диалоге
3. Пост будет удален с плавной анимацией
4. Статистика обновится автоматически

## Примеры использования

### Сценарий 1: Модерация платных постов
Администратор может быстро просмотреть все платные посты пользователя и деактивировать неподходящие, не удаляя их полностью.

### Сценарий 2: Удаление нарушающего контента
Если платный пост нарушает правила, администратор может его удалить, при этом связанные фото и альбомы останутся доступными.

### Сценарий 3: Проверка активности
Администратор может увидеть, сколько активных и неактивных платных постов у пользователя, и принять решение о дальнейших действиях.

## Технические детали

### Структура базы данных

**Таблица commercial_posts:**
- `id` - уникальный идентификатор
- `user_id` - ID пользователя
- `type` - тип поста (photo/album/standalone)
- `album_id` - ID альбома (если type = album)
- `photo_id` - ID фото (если type = photo)
- `title` - название
- `description` - описание
- `image_url` - URL изображения (для standalone)
- `latitude` - широта
- `longitude` - долгота
- `is_active` - статус (0/1)
- `created_at` - дата создания

### JavaScript функции

**loadCommercialPosts()** - загружает список платных постов  
**displayCommercialPosts(posts)** - отображает посты в интерфейсе  
**toggleCommercialStatus(id, status)** - меняет статус поста  
**deleteCommercialPost(id)** - удаляет платный пост
