# Requirements Document

## Introduction

Административная панель для управления контентом и пользователями приложения TravelMap. Панель должна предоставлять полный обзор активности пользователей, включая лайки, комментарии, подписки, избранное, публикации и модерацию контента.

## Glossary

- **Admin Panel**: Веб-интерфейс для администраторов системы
- **User**: Пользователь приложения TravelMap
- **Post**: Фотография с локацией, опубликованная пользователем
- **Commercial Post**: Коммерческое объявление, привязанное к альбому, фото или standalone
- **Album**: Коллекция фотографий пользователя
- **Like**: Отметка "нравится" на посте
- **Comment**: Текстовый комментарий к посту или альбому
- **Follow**: Подписка одного пользователя на другого
- **Favorite**: Добавление поста или альбома в избранное
- **Moderation**: Процесс проверки и удаления контента

## Requirements

### Requirement 1: Управление лайками

**User Story:** Как администратор, я хочу видеть все лайки в системе, чтобы отслеживать активность пользователей

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Лайки", THE Admin Panel SHALL отобразить список всех лайков с информацией о пользователе, который поставил лайк, и посте, которому поставлен лайк
2. THE Admin Panel SHALL отображать для каждого лайка: имя пользователя, фото профиля, название поста, дату создания лайка
3. THE Admin Panel SHALL предоставлять фильтрацию лайков по пользователю и по посту
4. THE Admin Panel SHALL предоставлять поиск лайков по имени пользователя
5. THE Admin Panel SHALL отображать общее количество лайков для каждого поста

### Requirement 2: Управление комментариями

**User Story:** Как администратор, я хочу видеть все комментарии в системе, чтобы модерировать контент

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Комментарии", THE Admin Panel SHALL отобразить список всех комментариев к постам и альбомам
2. THE Admin Panel SHALL отображать для каждого комментария: автора, текст комментария, пост/альбом, дату создания
3. THE Admin Panel SHALL предоставлять возможность удаления комментария
4. THE Admin Panel SHALL предоставлять фильтрацию комментариев по пользователю и по посту/альбому
5. THE Admin Panel SHALL предоставлять поиск комментариев по тексту

### Requirement 3: Управление пользователями

**User Story:** Как администратор, я хочу видеть детальную информацию о пользователях, чтобы анализировать их активность

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Пользователи", THE Admin Panel SHALL отобразить список всех пользователей с основной информацией
2. THE Admin Panel SHALL отображать для каждого пользователя: имя, email, фото профиля, дату регистрации, количество подписчиков, количество подписок
3. WHEN администратор выбирает пользователя, THE Admin Panel SHALL отобразить детальную страницу с полной информацией о пользователе
4. THE Admin Panel SHALL отображать на странице пользователя список его подписчиков
5. THE Admin Panel SHALL отображать на странице пользователя список пользователей, на которых он подписан
6. THE Admin Panel SHALL отображать на странице пользователя список постов, которые он добавил в избранное
7. THE Admin Panel SHALL отображать на странице пользователя список альбомов, которые он добавил в избранное
8. THE Admin Panel SHALL отображать на странице пользователя список постов, которые он комментировал
9. THE Admin Panel SHALL отображать на странице пользователя список постов, на которые ему оставили комментарии

### Requirement 4: Управление подписками

**User Story:** Как администратор, я хочу видеть все подписки в системе, чтобы анализировать социальные связи

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Подписки", THE Admin Panel SHALL отобразить список всех подписок
2. THE Admin Panel SHALL отображать для каждой подписки: подписчика, пользователя на которого подписались, дату подписки
3. THE Admin Panel SHALL предоставлять фильтрацию подписок по пользователю
4. THE Admin Panel SHALL предоставлять поиск подписок по имени пользователя

### Requirement 5: Управление избранным

**User Story:** Как администратор, я хочу видеть все добавления в избранное, чтобы понимать популярный контент

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Избранное", THE Admin Panel SHALL отобразить список всех добавлений в избранное
2. THE Admin Panel SHALL отображать избранное для постов, альбомов и коммерческих постов раздельно
3. THE Admin Panel SHALL отображать для каждого избранного: пользователя, контент, дату добавления
4. THE Admin Panel SHALL предоставлять фильтрацию избранного по типу контента и пользователю
5. THE Admin Panel SHALL отображать общее количество добавлений в избранное для каждого контента

### Requirement 6: Управление публикациями пользователей

**User Story:** Как администратор, я хочу видеть все публикации пользователей, чтобы контролировать контент

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Публикации", THE Admin Panel SHALL отобразить список всех постов, коммерческих постов и альбомов
2. THE Admin Panel SHALL отображать для каждого поста: заголовок, описание, локацию, автора, дату создания, превью изображения
3. THE Admin Panel SHALL отображать для каждого коммерческого поста: заголовок, описание, локацию, автора, тип (album/photo/standalone), связанные посты/альбомы
4. THE Admin Panel SHALL отображать для каждого альбома: название, описание, автора, количество фотографий, обложку
5. WHEN администратор выбирает альбом, THE Admin Panel SHALL отобразить список всех фотографий в альбоме
6. THE Admin Panel SHALL предоставлять фильтрацию публикаций по типу и автору
7. THE Admin Panel SHALL предоставлять поиск публикаций по заголовку и локации

### Requirement 7: Модерация контента

**User Story:** Как администратор, я хочу модерировать фотографии пользователей, чтобы удалять неподходящий контент

#### Acceptance Criteria

1. WHEN администратор открывает раздел "Модерация", THE Admin Panel SHALL отобразить все фотографии пользователей в виде галереи
2. THE Admin Panel SHALL отображать для каждой фотографии: превью, автора, дату загрузки, связанный пост/альбом
3. THE Admin Panel SHALL предоставлять возможность удаления фотографии
4. WHEN администратор удаляет фотографию, THE Admin Panel SHALL удалить фотографию из базы данных и файловой системы
5. THE Admin Panel SHALL предоставлять фильтрацию фотографий по автору и дате
6. THE Admin Panel SHALL предоставлять массовое удаление выбранных фотографий

### Requirement 8: Просмотр связей коммерческих постов

**User Story:** Как администратор, я хочу видеть в каких постах размещены коммерческие объявления, чтобы контролировать рекламу

#### Acceptance Criteria

1. WHEN администратор просматривает коммерческий пост, THE Admin Panel SHALL отобразить список всех постов, в которых он размещен
2. THE Admin Panel SHALL отображать для каждого связанного поста: превью, заголовок, локацию
3. WHEN коммерческий пост привязан к альбому, THE Admin Panel SHALL отобразить информацию об альбоме и список фотографий в нем
4. WHEN коммерческий пост привязан к фото, THE Admin Panel SHALL отобразить информацию о фото и его локации
5. THE Admin Panel SHALL отображать тип коммерческого поста (album/photo/standalone)
