# Проверка отображения даты рождения

## Почему дата рождения не показывается?

Есть несколько возможных причин:

### 1. Колонка не существует в базе данных
Колонка `date_of_birth` может отсутствовать в таблице `users`.

**Проверка:**
```bash
# Откройте в браузере:
test_user_birth_date.php?user_id=27
```

**Решение:**
```bash
# Откройте в браузере:
apply_user_auth_migration.php
```

Или выполните SQL вручную:
```sql
ALTER TABLE users 
ADD COLUMN date_of_birth DATE NULL AFTER phone_number;
```

### 2. У пользователя нет даты рождения
Значение `date_of_birth` может быть `NULL` в базе данных.

**Проверка:**
```sql
SELECT id, first_name, last_name, date_of_birth 
FROM users 
WHERE id = 27;
```

**Решение:**
Добавьте дату рождения пользователю:
```sql
UPDATE users 
SET date_of_birth = '1990-01-01' 
WHERE id = 27;
```

### 3. API не возвращает дату рождения
API может не включать поле `dateOfBirth` в ответ.

**Проверка:**
```bash
# Откройте в браузере:
api/users/get_user_details.php?user_id=27
```

Найдите в JSON ответе:
```json
{
  "user": {
    "dateOfBirth": "1990-01-01"  // Должно быть здесь
  }
}
```

### 4. JavaScript не обрабатывает дату
Может быть ошибка в JavaScript коде.

**Проверка:**
1. Откройте консоль браузера (F12)
2. Перейдите на страницу пользователя
3. Найдите в консоли:
```
Date of birth: 1990-01-01
```

Если видите:
- `Date of birth: null` - у пользователя нет даты рождения
- `Date of birth: undefined` - API не возвращает поле
- `Invalid birth date` - дата в неправильном формате
- `No birth date available` - дата пустая или null

## Пошаговая проверка

### Шаг 1: Проверьте структуру таблицы
```bash
# Откройте:
check_users_table.php
```

Должна быть колонка `date_of_birth`.

### Шаг 2: Проверьте данные пользователя
```bash
# Откройте:
test_user_birth_date.php?user_id=27
```

Должно показать дату рождения пользователя.

### Шаг 3: Проверьте API
```bash
# Откройте:
api/users/get_user_details.php?user_id=27
```

Найдите `"dateOfBirth"` в JSON ответе.

### Шаг 4: Проверьте консоль браузера
1. Откройте страницу пользователя
2. Откройте консоль (F12)
3. Найдите сообщение `Date of birth:`

## Быстрое решение

Если колонка не существует:
```bash
# Откройте в браузере:
apply_user_auth_migration.php
```

Если у пользователя нет даты:
```sql
-- Добавьте тестовую дату
UPDATE users 
SET date_of_birth = '1990-01-01' 
WHERE id = 27;
```

Затем обновите страницу пользователя.

## Ожидаемый результат

После исправления вы должны увидеть:
```
Дата рождения: 01.01.1990 (34 года)
```

## Отладка

### В консоли браузера должно быть:
```
Date of birth: 1990-01-01
```

### Если видите ошибку:
```
Error parsing birth date: ...
```

Значит дата в неправильном формате. Проверьте формат в базе данных (должен быть DATE или YYYY-MM-DD).

## Формат даты

### В базе данных:
- Тип: `DATE`
- Формат: `YYYY-MM-DD` (например: `1990-01-01`)

### В API:
- Формат: `YYYY-MM-DD` (например: `"1990-01-01"`)

### На странице:
- Формат: `ДД.ММ.ГГГГ (возраст лет)` (например: `01.01.1990 (34 года)`)

## Проверка работы

1. Откройте `test_user_birth_date.php?user_id=27`
2. Убедитесь что колонка существует
3. Убедитесь что у пользователя есть дата
4. Откройте страницу пользователя
5. Проверьте консоль браузера
6. Дата должна отображаться
