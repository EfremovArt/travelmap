# Улучшение отображения комментариев в модерации

## Проблемы

1. **Не отображается информация о посте** - если у поста нет названия (`photo_title` пустой или null), информация о посте не показывалась
2. **Нет переносов строк** - комментарии с переносами строк отображались в одну строку
3. **Непонятно, что пост удалён** - если пост был удалён, не было никакого указания на это

## Решения

### 1. Отображение постов без названия

**Было:**
```javascript
${comment.photo_id && comment.photo_title ? `...` : ''}
```

**Стало:**
```javascript
${comment.photo_id ? `
    К посту: ${escapeHtml(comment.photo_title || 'Без названия')}
` : ''}
```

Теперь пост отображается даже если у него нет названия.

### 2. Поддержка переносов строк

Добавлен стиль `white-space: pre-wrap` для текста комментария:
```html
<p class="mb-2" style="white-space: pre-wrap;">${escapeHtml(comment.text)}</p>
```

Теперь переносы строк в комментариях сохраняются.

### 3. Индикация удалённых постов

Если у комментария нет ни `photo_id`, ни `album_id`, показывается сообщение:
```html
<small class="text-muted">
    <i class="bi bi-exclamation-triangle"></i> Пост удалён или недоступен
</small>
```

### 4. Fallback для отсутствующего превью

Если у поста нет превью изображения, показывается иконка:
```html
${comment.photo_preview ? `
    <img src="${photoPreview}" ...>
` : `
    <i class="bi bi-image me-2" style="font-size: 2rem; color: #3498db;"></i>
`}
```

## Результат

✅ Все комментарии теперь показывают информацию о посте/альбоме  
✅ Переносы строк в комментариях сохраняются  
✅ Понятно, когда пост был удалён  
✅ Есть fallback для отсутствующих превью
