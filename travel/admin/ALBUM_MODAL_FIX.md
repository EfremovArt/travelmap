# Исправление открытия альбомов в модальном окне

## Проблема
При клике на комментарий к альбому в разделе модерации происходило перенаправление на страницу "Управление публикациями", вместо открытия альбома в модальном окне.

## Решение

### 1. JavaScript (moderation.js)

Обновлена функция `viewAlbumDetailsFromComment`:

**Было:**
```javascript
function viewAlbumDetailsFromComment(albumId) {
    window.location.href = `posts.php?album_id=${albumId}`;
}
```

**Стало:**
```javascript
function viewAlbumDetailsFromComment(albumId) {
    // Загружает детали альбома через API
    // Отображает в модальном окне с фотографиями
}
```

### 2. Новая функция `showAlbumPreview`

Создана функция для отображения альбома в модальном окне:
- Показывает название и описание альбома
- Отображает информацию об авторе и дате создания
- Показывает количество фотографий и комментариев
- Отображает все фотографии альбома в виде сетки
- При клике на фото открывается детальный просмотр

### 3. API (get_all_albums.php)

Добавлена поддержка фильтра по `album_id`:
```php
$albumId = isset($_GET['album_id']) ? intval($_GET['album_id']) : 0;

if ($albumId > 0) {
    $whereConditions[] = "a.id = :album_id";
    $params[':album_id'] = $albumId;
}
```

## Как работает

1. Пользователь кликает на комментарий к альбому
2. Вызывается `viewAlbumDetailsFromComment(albumId)`
3. Загружаются детали альбома через `get_all_albums.php?album_id=X`
4. Загружаются фотографии альбома через `get_album_photos.php?album_id=X`
5. Отображается модальное окно с информацией и фотографиями
6. При клике на фото открывается детальный просмотр поста

## Модальное окно показывает:

- **Заголовок:** Название альбома
- **Описание:** Если есть
- **Информация:**
  - Автор альбома
  - Дата создания
  - Количество фотографий
  - Количество комментариев
- **Фотографии:** Сетка с превью всех фотографий (кликабельные)

## Изменённые файлы

1. `travel/admin/assets/js/moderation.js` - функции просмотра альбома
2. `travel/admin/api/posts/get_all_albums.php` - поддержка фильтра по album_id

## Исправленные ошибки

### Ошибка 1: `TypeError: can't access property "textContent", modalTitle is null`

**Причина:** Неправильный ID элемента - использовался `photoPreviewModalLabel`, а существует `photoPreviewTitle`

**Решение:** 
- Исправлен ID элемента на `photoPreviewTitle`
- Добавлена проверка существования модального окна
- Добавлена проверка существования элемента заголовка

### Ошибка 2: `photo_id=undefined` в запросах

**Причина:** API `get_album_photos.php` возвращает поле `photo_id`, а не `id`

**Решение:**
- Обновлен код для использования правильного поля: `photo.photo_id || photo.id`
- Теперь клик по фотографии корректно открывает детальный просмотр

### Ошибка 3: `TypeError: can't access property "src", document.getElementById(...) is null`

**Причина:** После показа альбома структура модального окна изменялась, и элементы `photoPreviewImage` и `photoPreviewInfo` удалялись

**Решение:**
- Добавлена проверка и восстановление оригинальной структуры модального окна в функции `showPhotoPreview`
- Теперь модальное окно корректно переключается между показом альбома и фото

### Ошибка 4: Остаётся backdrop (затемнение) после закрытия модального окна

**Причина:** Создавались множественные экземпляры `bootstrap.Modal`, что приводило к накоплению backdrop элементов

**Решение:**
- Используется `bootstrap.Modal.getInstance()` для переиспользования существующего экземпляра
- Добавлена очистка всех backdrop элементов при закрытии модального окна
- Удаляется класс `modal-open` с body и сбрасываются стили overflow и padding

## Тестирование

1. Откройте Модерация → Комментарии
2. Найдите комментарий к альбому (с иконкой коллекции)
3. Кликните на "К альбому: [название]"
4. Должно открыться модальное окно с деталями альбома
5. Проверьте, что отображаются все фотографии
6. Кликните на любую фотографию - должен открыться детальный просмотр
